# Get Context

Use this before Product Design design, ideation, clone, prototype, redesign, or image-to-code work.

Gather only what is missing:

1. What product, screen, component, workflow, or feature is being designed?
2. What visual source should guide it: URL, screenshot, mockup, Figma, local app, existing design
   system, saved Product Design context, or a named style direction?
3. What interactivity level is expected?
   - `full`: controls, states, and core interactions work.
   - `static`: enough interaction for review, optimized for speed.

If the answers are already present, do not ask again. Play back a short design brief and name the
next step. Continue only after the user confirms the brief, unless the same brief was already
confirmed in the thread.

For involved builds, after confirmation tell the user the work may take several minutes and that
you will keep moving.

Do not scaffold, edit files, run image generation, or start a server while required context is
missing.
