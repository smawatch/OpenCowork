# Share

Use this when the user asks to share, publish, deploy, host, or create a link for a runnable
prototype.

## Workflow

1. Confirm the prototype directory and deployment target.
2. If the user named a configured MCP or deployment tool, use it.
3. If no target is clear, ask where to deploy.
4. Build the prototype if the target requires build output.
5. Deploy with the selected tool when available.
6. Return the working share URL.

Do not claim the prototype is shared until there is a working URL. If the selected target is not
available, say so and ask for another target.
