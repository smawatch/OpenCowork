# Research

Use this for current, source-grounded research into UX friction, onboarding pain, docs/help issues,
support pain, developer experience friction, or product workflow problems for a named digital
product.

## Workflow

1. Restate product, audience, time horizon, and research scope.
2. Search public sources with `WebSearch` when enabled: Reddit, X/Twitter, Hacker News, Stack
   Overflow, GitHub issues/discussions, forums, blogs, reviews, and relevant communities.
3. Search internal sources only when the user authorizes them and tools are configured.
4. Cluster evidence into high-signal problems.
5. Separate UX friction, docs/help friction, onboarding, account/billing/permissions, developer/API
   friction, reliability/performance, and feature requests.
6. Rank by severity, frequency signal, confidence, and product leverage.

## Output

Return an in-chat research brief unless the user asks for another format:

- executive read
- ranked UX problems with evidence and confidence
- source map
- opportunity map: fix this week, fix this quarter, needs deeper research

Cite sources when available. Do not overclaim from anecdotes.
