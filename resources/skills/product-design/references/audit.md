# Audit

Use this for UX, design, or accessibility audits of a product flow, journey, screen, or bounded
interaction.

## Workflow

1. Identify the product, target flow, audit scope, destination, and capture tool.
2. If destination is missing, ask whether to save output in a local folder or a configured design
   tool/MCP target.
3. Capture every important step with Browser tools. Save screenshots with ordered names such as
   `01-start.png`, `02-form-filled.png`, `03-confirmation.png`.
4. Inspect each saved screenshot before accepting it. Reject wrong, blank, cropped, blocked, or
   loading screenshots.
5. For each step, write notes tied to evidence:
   - strengths
   - UX risks
   - accessibility risks
   - evidence limits
   - concrete recommendations
6. Save notes beside the screenshots or place them in the selected destination.

Do not claim full WCAG compliance from screenshots alone. Distinguish visible evidence from
inference.

Final response: list each captured step, its general health, and where the full audit output was
saved.
