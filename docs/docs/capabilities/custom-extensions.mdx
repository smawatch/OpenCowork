# 自定义插件 / Custom Extensions

Extensions 是 OpenCowork 的用户自定义插件体系。它和 **App plugins**、**消息平台插件** 是三套不同表面：

| 表面            | 面向谁              | 主要能力                                      |
| --------------- | ------------------- | --------------------------------------------- |
| Extensions      | 用户和团队本地扩展  | 自定义 Agent 工具、响应 UI、受限 HTTP/JS 执行 |
| App plugins     | OpenCowork 内置能力 | 图像、浏览器、桌面控制                        |
| Channel plugins | 消息平台接入        | 飞书、钉钉、Telegram、Discord 等消息收发      |

V1 的目标是“简单、可安装、可审计”。一个 extension 是一个本地文件夹，包含 `extension.json`、可选 JS entry 和可选 HTML renderer。

## 安装位置 / Install location

安装后文件会复制到：

```text
~/.open-cowork/extensions/<extensionId>/extension.json
```

状态文件保存在：

```text
~/.open-cowork/extensions.json
~/.open-cowork/extensions-storage.json
```

`secret` 配置写入 OpenCowork 的安全配置存储，普通配置写入 `extensions.json`。

安装入口：

1. 打开 **Settings → Extensions / 自定义插件**
2. 点击 **Install folder**
3. 选择一个包含 `extension.json` 的文件夹
4. 安装后插件默认 disabled，需要手动启用

## 最小目录 / Minimal folder

```text
my-extension/
  extension.json
  index.js
  renderer.html
```

## Manifest

`extension.json` 是插件的唯一声明入口：

```json
{
  "schemaVersion": 1,
  "id": "demo_extension",
  "name": "Demo Extension",
  "version": "0.1.0",
  "description": "Demo custom tools and response UI.",
  "entry": "index.js",
  "permissions": {
    "network": ["https://jsonplaceholder.typicode.com"]
  },
  "tools": [
    {
      "name": "get_post",
      "description": "Fetch a demo post by id.",
      "kind": "http",
      "inputSchema": {
        "type": "object",
        "properties": {
          "id": { "type": "string", "description": "Post id" }
        },
        "required": ["id"]
      },
      "http": {
        "method": "GET",
        "url": "https://jsonplaceholder.typicode.com/posts/{{input.id}}"
      }
    },
    {
      "name": "show_table",
      "description": "Return a table rendered in chat.",
      "kind": "js",
      "handler": "showTable",
      "readOnly": true,
      "inputSchema": {
        "type": "object",
        "properties": {}
      }
    }
  ],
  "renderers": [
    {
      "name": "summary_card",
      "type": "html",
      "entry": "renderer.html"
    }
  ]
}
```

工具名会自动注册为：

```text
extension__demo_extension__get_post
extension__demo_extension__show_table
```

这个前缀避免和内置 tools、MCP tools、消息平台 tools 冲突。

## Manifest 字段 / Manifest fields

| 字段                  | 必填                | 说明                                         |
| --------------------- | ------------------- | -------------------------------------------- |
| `schemaVersion`       | 是                  | V1 固定为 `1`                                |
| `id`                  | 是                  | 小写字母、数字、`_`、`-`，安装目录名必须一致 |
| `name`                | 是                  | 设置页展示名                                 |
| `version`             | 是                  | 插件版本                                     |
| `description`         | 否                  | 设置页说明                                   |
| `entry`               | JS 工具需要         | sandbox JS 文件                              |
| `configSchema`        | 否                  | 用户可配置字段，支持 `text` / `secret`       |
| `permissions.network` | HTTP/ctx.fetch 需要 | origin allowlist，未声明则不能联网           |
| `tools`               | 是                  | 工具列表                                     |
| `renderers`           | 否                  | HTML 响应 UI renderer                        |

配置字段示例：

```json
{
  "configSchema": [
    { "key": "baseUrl", "label": "Base URL", "type": "text", "required": true },
    { "key": "apiKey", "label": "API Key", "type": "secret", "required": true }
  ]
}
```

字符串字段支持插值：

```text
{{input.foo}}
{{input.user.id}}
{{config.baseUrl}}
```

HTTP tool 的 `url`、`headers`、`body` 都会递归插值。

## HTTP 工具 / HTTP tools

HTTP 工具是声明式工具，不需要写 JS：

```json
{
  "name": "search",
  "description": "Search the company API.",
  "kind": "http",
  "inputSchema": {
    "type": "object",
    "properties": {
      "query": { "type": "string" }
    },
    "required": ["query"]
  },
  "http": {
    "method": "GET",
    "url": "{{config.baseUrl}}/search?q={{input.query}}",
    "headers": {
      "Authorization": "Bearer {{config.apiKey}}"
    }
  }
}
```

所有网络请求都走 main process 的 `extension:fetch` IPC，并按 `permissions.network` 校验。allowlist 不匹配时，工具会返回权限错误。

## JS 工具 / JS tools

JS 工具运行在隐藏 iframe 沙箱里：

- `sandbox="allow-scripts"`
- CSP：`default-src 'none'; script-src 'unsafe-inline'; style-src 'unsafe-inline'; img-src data:`
- 不允许 Node import
- 不暴露 Electron、文件系统、Shell、Node API
- 直接 `fetch()` 会被禁用
- 只能通过 `ctx.fetch`、`ctx.storage`、`ctx.config` 与宿主交互

`index.js` 必须设置 `globalThis.openCoworkExtension`：

```js
globalThis.openCoworkExtension = {
  handlers: {
    async showTable(input, ctx) {
      return {
        text: 'Here is a table from a sandboxed extension.',
        data: [
          { name: 'Alpha', value: 3 },
          { name: 'Beta', value: 7 }
        ],
        ui: {
          kind: 'table',
          columns: ['name', 'value'],
          rows: [
            { name: 'Alpha', value: 3 },
            { name: 'Beta', value: 7 }
          ]
        }
      }
    }
  }
}
```

使用 `ctx.fetch`：

```js
async function getPost(input, ctx) {
  const response = await ctx.fetch({
    method: 'GET',
    url: `https://jsonplaceholder.typicode.com/posts/${input.id}`
  })

  return {
    text: `HTTP ${response.status}`,
    data: response.json ?? response.text
  }
}
```

使用 `ctx.storage`：

```js
async function remember(input, ctx) {
  await ctx.storage.set('last_query', input.query)
  return { text: `Saved ${input.query}` }
}
```

## 工具结果 / Tool result

Extension 工具结果会编码成现有 structured tool result，不改变 provider 协议：

```ts
{
  __openCoworkExtensionResult: true,
  extensionId: 'demo_extension',
  toolName: 'show_table',
  text: '...',
  data: {},
  ui: { kind: 'table', rows: [] }
}
```

Agent 仍然收到普通 tool result，聊天 UI 会识别 `__openCoworkExtensionResult` 并渲染更友好的结果卡片。

## 响应 UI / Response UI

内置 schema renderer 支持：

| `ui.kind` | 用途                             |
| --------- | -------------------------------- |
| `card`    | 标题、正文、小列表               |
| `table`   | 表格数据                         |
| `form`    | 只读表单                         |
| `chart`   | 简单条形图                       |
| `html`    | 插件提供的 sandbox HTML renderer |

### Table

```js
return {
  text: 'Top results',
  ui: {
    kind: 'table',
    columns: ['name', 'score'],
    rows: [
      { name: 'Alpha', score: 0.93 },
      { name: 'Beta', score: 0.81 }
    ]
  }
}
```

### HTML renderer

Manifest：

```json
{
  "renderers": [{ "name": "summary_card", "type": "html", "entry": "renderer.html" }]
}
```

工具返回：

```js
return {
  text: 'Rendered by custom HTML.',
  ui: {
    kind: 'html',
    renderer: 'summary_card',
    props: {
      title: 'Demo',
      rows: [{ label: 'Status', value: 'Ready' }]
    }
  }
}
```

`renderer.html`：

```html
<div id="root"></div>
<script>
  window.addEventListener('extension-props', (event) => {
    const props = event.detail
    document.getElementById('root').innerHTML = `
      <section style="padding:12px;border:1px solid #64748b;border-radius:8px">
        <strong>${props.title}</strong>
        <pre>${JSON.stringify(props.rows, null, 2)}</pre>
      </section>
    `
  })
</script>
```

HTML renderer 也运行在 `sandbox="allow-scripts"` iframe 中，不能访问父页面 DOM。

## 审批策略 / Approval policy

| 工具类型               | 默认审批   |
| ---------------------- | ---------- |
| HTTP `GET`             | 不需要审批 |
| HTTP 非 `GET`          | 需要审批   |
| JS                     | 需要审批   |
| 显式 `readOnly: true`  | 不需要审批 |
| 显式 `readOnly: false` | 需要审批   |

建议保守声明：只给纯读取工具设置 `readOnly: true`。

## 调试 / Debugging

1. 从 Settings → Extensions 安装示例目录。
2. 启用 extension。
3. 打开任意聊天，询问 Agent 使用对应工具。
4. 如果工具没有出现，重新发起一次请求；OpenCowork 会在请求前刷新 dynamic tool catalog。
5. 如果网络报错，检查 `permissions.network` 是否包含目标 origin。
6. 如果 JS handler 报错，确认 `entry` 文件存在，且 handler 名和 manifest 的 `handler` 一致。

## 示例插件 / Example extension

仓库内提供了一个可安装示例：

```text
examples/extensions/demo-extension/
```

它包含：

- 声明式 HTTP GET 工具
- JS handler
- `ui.kind = 'table'`
- `ui.kind = 'html'` sandbox renderer

你也可以先运行 smoke test：

```bash
node scripts/verify-extension-example.mjs
```

开发时可以复制这个目录，然后改 `id`、`name`、`tools` 和 handler。

## V1 边界 / V1 limits

V1 不支持：

- 自定义消息平台 provider
- 插件市场
- zip / remote install
- React 组件包
- 插件 JS 直接访问文件系统、Shell、Electron 或 Node

需要这些能力时，应通过后续宿主 RPC 设计，而不是绕过沙箱。
